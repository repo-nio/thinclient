Introduction

Le Superviseur Nixxis vous donne les moyens d'obtenir et de maintenir une interaction client optimale grâce à une interface facile à utiliser. Le Superviseur Nixxis vous offre tous les outils pour y parvenir.

-      Il est facile à configurer et vous permet de voir exactement ce dont vous avez besoin.

-      Outils pour vous aider à coacher vos agents et améliorer l'interaction client

-      Vues de tous les aspects importants de la Nixxis Contact Suite :

o    Vue agent.

o    Vue équipes

o    Vue entrante et sortante.

o    Vue file d'attente.

Dans ce document, nous allons passer en revue les différentes notions dont vous avez besoin pour travailler avec le Superviseur Nixxis. En outre, nous y expliquerons comment configurer et utiliser les outils fournis par le Superviseur Nixxis. Dans la partie suivante, nous décrivons toutes les vues ainsi que les données qu'elles fournissent. A la fin du document, l'annexe fournit un complément d'informations à propos des principes que la Nixxis Contact Suite utilise dans ses relations avec le Superviseur Nixxis.

Alors allons-y !

 

Premiers pas avec le Superviseur

Aperçu

Ce chapitre décrit comment accéder au Superviseur Nixxis et comment utiliser l'écran de navigation.

Démarrage du Superviseur Nixxis

Le Superviseur Nixxis est une application web qui nécessite un navigateur. Il est compatible avec les navigateurs Internet Explorer, Firefox, Safari ou Opera. Lancez le navigateur de votre choix puis allez à la barre d'adresse et tapez l'URL de votre superviseur.

Par exemple :

Après avoir saisi l'URL correct, vous verrez apparaître l'écran suivant. Vous pouvez y saisir vos informations de logon.

Le superviseur doit à présent fournir ses informations de logon pour pouvoir commencer à utiliser l'application (voir : Guide d'administration Nixxis pour plus d'informations à propos de la création d'agents superviseurs).

Fournissez ensuite les informations de login suivantes :

-      User account : l'identifiant unique fourni par l'administrateur pour se connecter au système.

-      Password : le mot de passe de l'agent qui souhaite ouvrir une session.

-      Domain : destiné à une utilisation future.

Lorsque c'est fait, vous êtes prêt à ouvrir une session dans le système en cliquant sur le bouton . Vous avez aussi la possibilité de cliquer sur le bouton  pour rétablir la valeur par défaut de tous les champs. Une fois les informations de logon correctes, vous allez être dirigé vers la fenêtre de navigation du Superviseur Nixxis.

Travailler avec les vues du Superviseur Nixxis

Aperçu

Ce chapitre va expliquer comment les vues du Superviseur Nixxis fonctionnent et comment vous pouvez les configurer.

Vous trouverez au chapitre suivant "Description des colonnes" des informations détaillées sur les différentes colonnes qui peuvent être trouvées.

Configuration d'une fenêtre de vue

La configuration de base de toutes les vues est identique.

Cette fenêtre s'articule en trois zones principales :

  1. Barre d'outils : la zone indique les barres d'outils avec toutes les options que le superviseur peut utiliser dans la vue en question
  2. Vue principale : il s'agit de la zone dans laquelle la liste des vues sélectionnées s'affiche.
  3. Panneau d'information : fournit des informations supplémentaires sur une colonne sélectionnée de la vue principale.

Zone barre d'outils

Aperçu

Dans la zone barre d'outils, le superviseur a accès aux différentes options d'une vue. Un superviseur peut utiliser trois types de barres d'outils.

-       Barre d'outils du navigateur

-       Barre d'outils de configuration

-       Barre d'outils de coaching.

En fonction de la vue sélectionnée, les barres d'outils et options qui peuvent être utilisées avec cette vue sont rendues disponibles pour le superviseur. Par exemple, la 'barre d'outils de coaching' est uniquement disponible avec la vue agent parce que c'est la zone dans laquelle un agent peut être sélectionné.

Barre d'outils du navigateur

Aperçu

Il s'agit de la même barre d'outils que la barre d'outils du navigateur à la page 11. La vue courante ne va pas s'afficher dans la barre d'outils. Par exemple si la vue agent est la vue active, cela ne sera pas visible dans la barre d'outils du navigateur.

Barre d'outils de configuration

Aperçu

La barre d'outils de configuration est utilisée pour personnaliser la vue, ce qui permet au superviseur de sélectionner les paramètres qui l'intéressent.

Les options suivantes sont mise à la disposition du superviseur :

-          Une étiquette indiquant qu'il s'agit de la barre d'outils options de la vue. Elle n'est pas cliquable. L'icône sera différente pour chacune des vues.

o        Vue agent.

o        Vue entrée.

o        Vue sortie.

o        Vue file d'attente.

o        Vue équipes.

-          Montre ou masque le "Panneau d'information" (voir page 14 pour plus d'informations).

-          Sélection de la colonne (voir page 14 pour plus d'informations).

-          Sélectionner le champ regroupement par (voir page 15 pour plus d'informations).

-          Créer des colonnes définies par l'utilisateur (voir page 16 pour plus d'informations).

Sélection de colonne

Lorsque l'option ‘Sélection de colonne' est cliquée, une nouvelle fenêtre s'affiche. Elle permet au superviseur de sélectionner les colonnes qu'il désire voir apparaître.

Vous trouverez sur la droite un exemple de fenêtre "Sélection de colonne". Quand la fenêtre est ouverte pour la première fois, elle présente le statut courant des colonnes dans la "vue principale".

Dans la fenêtre, nous voyons apparaître trois colonnes :

-      Nom de la colonne.

-      Case à cocher :

o     La colonne n'est pas visible dans la vue principale.

o     La colonne est visible dans la vue principale.

-      Catégorie de colonne.

Si le superviseur souhaite visualiser une colonne, il lui suffit de cliquer sur la case à cocher et de s'assurer qu'elle est bien cochée ‘’. S'il ne souhaite pas voir apparaître la colonne, il doit décocher ‘’ la case à cocher. Le bouton ‘’ va mettre à jour la vue principale. Il peut également cliquer sur le bouton ‘’ pour appliquer le changement et fermer la fenêtre "Sélection de colonne".

Le bouton ‘’ va réinitialiser le changement effectué par le superviseur et la sélection courante va être restaurée. Si le superviseur clique sur le bouton ‘’, aucun changement ne sera apporté à la sélection de colonne et la fenêtre va se refermer. La fenêtre peut également être fermée en cliquant sur le bouton ‘’ situé dans le coin supérieur droit de la fenêtre. Cela ne va toutefois pas appliquer le changement que vous avez apporté à la sélection de colonne et ne réinitialisera pas la sélection que vous avez effectuée dans la fenêtre de sélection de colonne. La prochaine fois que vous ouvrirez la fenêtre, votre sélection sera donc celle que vous avez effectuée précédemment (pas de présentation de la sélection dans la vue principale).

Si toutes les colonnes ne peuvent pas être affichées dans la fenêtre de sélection de colonne, le superviseur peut soit reformater la fenêtre, soit utiliser la barre de défilement qui sera visible sur le côté droit de la fenêtre de sélection de colonne.

Dans la dernière colonne, vous pouvez voir la 'Catégorie de colonne'. Chacune des colonnes fournies par la Nixxis Contact Suite a été placée dans une catégorie. Le Superviseur Nixxis connaît les catégories suivantes :

-      Paramètres.

-      Temps réel.

-      Historique.

-      Colonnes systèmes.

-      Colonnes personnalisées (ou colonnes définies par l'utilisateur). Voir section 'Colonnes définies par l'utilisateur' à la page 16 pour plus d'informations concernant les 'Colonnes définies par l'utilisateur'.

Vous trouverez plus d'informations à propos des catégories à la page 26.

Vous trouverez des détails concernant les différentes colonnes des différentes vues au chapitre suivant 'Description des colonnes' à la page 26.

Fenêtre regroupement par

Si vous cliquez sur le bouton de regroupement de sélection ‘’, la fenêtre suivante va apparaître :

Dans cette fenêtre, le superviseur peut sélectionner le groupe qu'il souhaite visualiser et la colonne sur laquelle le regroupement doit s'effectuer dans la vue principale.

Un regroupement est utilisé pour lier des informations qui doivent être regroupées. Par exemple, nous pouvons regrouper tous les agents qui travaillent sur le helpdesk. Cela facilite la comparaison des résultats de rubriques qui sont faites pour être ensemble

Dans l'administrateur (voir manuel 4Principes généraux de l'Administrateur Nixxis4 pour plus d'informations au sujet de l'administrateur), nous pouvons lier un groupe à un agent. Ce groupe sera le champ de groupe par défaut du Superviseur Nixxis. Dans le Superviseur, vous pouvez cependant effectuer le regroupement sur n'importe quelle colonne disponible dans une vue.

Sélectionnez dans la liste déroulante la colonne sur laquelle vous souhaitez que le groupement soit réalisé. Si aucun regroupement n'est nécessaire, sélectionnez la première option ‘-Aucun-’ dans la liste. Une fois la sélection effectuée, cliquez sur le bouton ‘’ pour appliquer le changement à la vue principale. Cliquez sur le bouton ‘’ ou sur le bouton  dans le coin supérieur droit de la fenêtre pour refermer la fenêtre.

Les lignes qui n'ont pas de valeur pour la colonne sélectionnée ou dont la valeur est nulle adopteront la rubrique du groupe par défaut avec la description "Défaut".

La liste de regroupement est un tableau qui indique toutes les descriptions de groupe connues. Il indique également si le groupe est déployé ou réduit.

Voici à quoi ressemble une rubrique de groupe dans la liste :

Chaque rubrique de regroupement présente une description et peut être réduite ou déployée. La description d'une rubrique de regroupement est précédée d'une icône montrant si le groupe est réduit ou déployé. L'icône ‘’ vous indique que le groupe est réduit. Si vous cliquez sur cette icône, le groupe va se déployer et l'icône va se transformer en ‘’. Si vous cliquez sur cette icône, le groupe va à nouveau se réduire.

Voici à quoi ressemble le regroupement dans la vue principale :

Si aucune rubrique ne doit être affichée sous une rubrique de regroupement, cette rubrique ne sera pas affichée dans la vue principale.

Il existe deux façons de regrouper :

-      Regroupement de colonnes avec "valeurs ID" : certaines colonnes, comme la colonne ‘teamid’ ou la vue agent, sont des listes d'identifiants. Dans ce cas, le regroupement pour ces colonnes est configuré pour utiliser la valeur de la description dans la liste correspondante (dans notre exemple la description de la liste équipe). Cela signifie également que la ligne de la liste sera affichée dans toutes ses rubriques de groupe pour lesquelles il y a un ID.  Donc, si l'agent est un membre de deux équipes, il sera affiché dans ces deux rubriques de groupe.

-      Regroupement de colonnes avec "valeurs" : la valeur de la colonne est la rubrique de groupe. S'il n'y a pas de valeur, elle adoptera la rubrique de groupe par défaut.

Pour savoir comment une colonne sera regroupée, allez au chapitre 'Description des colonnes' à la page 26.

Colonnes définies par l'utilisateur

L'utilisateur peut accéder à la fenêtre utilisée pour créer des "Colonnes définies par l'utilisateur" en cliquant sur le bouton ‘’.

La fenêtre 'Colonnes définies par l'utilisateur' est utilisée pour créer des colonnes définies par le superviseur. Cela offre au superviseur un outil puissant pour personnaliser le Superviseur Nixxis et gérer sa Nixxis Contact Suite.

Une colonne définie par l'utilisateur est une colonne dans laquelle le superviseur peut choisir les informations affichées. Ces informations peuvent consister en un calcul des colonnes existantes. Les données peuvent être numériques, textuelles ou une combinaison des deux.

Quand la fenêtre est ouverte, nous pouvons voir apparaître trois zones différentes. La première partie est la zone où une sélection peut être effectuée. Cliquez sur le bouton ‘’ pour ajouter une nouvelle colonne. Utilisez la liste déroulante pour sélectionner une colonne existante. Ensuite, cliquez sur le bouton ‘’ pour éditer la sélection ou sur le bouton ‘’ pour supprimer la colonne.

La seconde partie est la zone dans laquelle la nouvelle colonne peut être configurée ou dans laquelle la colonne existante peut être éditée.

-      Name : le nom de la colonne est aussi le nom qui s'affiche dans l'en-tête de colonne.

-      Description : la description de la colonne.

-      Style : la façon dont les cellules vont être affichées. Les options sélectionnables sont texte, numérique, pourcentage et durée. Vous trouverez plus d'informations dans la section ‘Paramètres d'une colonne' à la page 21.

-      Visible : cette nouvelle colonne est visible par défaut dans la vue principale. Pour plus d'informations sur la vue principale, allez à la section 'Zone principale' à la page 19.

-      Visible in panel : cette colonne est visible dans le "Panneau d'information" lorsque le type de panneau est "Panneau détails". Reportez-vous pour plus d'informations à la section 'Panneau détails' à la page 23.

Dans la grande zone de texte située sous la configuration, le superviseur peut saisir la valeur de la nouvelle colonne. Cette valeur peut être statique, dynamique ou une combinaison des deux. Le superviseur peut donc choisir d'utiliser des fonctions disponibles en langage JavaScript (pour plus d'informations à ce propos, allez à la section 'Le langage JavaScript' à la page 42).

Le Superviseur Nixxis fournit une fonction appelée "ce. ([Js name])" qui permet d'accéder aux colonnes existantes (voir les chapitres individuels consacrés à la vue pour plus d'informations à propos des 'Js names’ que vous pouvez utiliser).

Par exemple : this.GetValueOf(Agent.Parameters.Firstname());
Cette fonction permet de revenir au prénom de l'agent (plus d'informations à propos de cette fonction à la page 44).

Barre d'outils de coaching.

Aperçu

La "Barre d'outils de coaching" contient des outils permettant de surveiller les agents de la Nixxis Contact Suite. Cela peut aider le superviseur à atteindre une interaction optimale avec le client.

Les options suivantes sont disponibles :

-          Une étiquette indiquant qu'il s'agit de la "barre d'outils de coaching". Elle n'est pas cliquable.

-          Pour utiliser la fonction 'écouter'' (voir 'Ecouter’ à la page 19).

-          La fonction 'intrusion' (voir ‘Intrusion’ à la page 19).

-      ou    démarrer ou arrêter d'enregistrer une conversation (voir ‘Enregistrement’ à la page 19).

Ecouter

La fonction d'écoute peut être utilisée par le superviseur pour écouter la conversation de l'agent.

Pour ce faire, le superviseur doit d'abord sélectionner un agent dans la liste, puis enfoncer le bouton ‘’ pour activer l'écoute. S'il souhaite mettre fin à l'écoute, le superviseur doit cliquer à nouveau sur ce bouton.

Intrusion

La fonction d'intrusion permet au superviseur d'intervenir dans une conversation de l'agent. Cela signifie que le superviseur peut participer à une conversation entre l'agent et le client. Lorsque le superviseur utilise cette fonction, toutes les parties peuvent s'entendre et se parler.

Pour utiliser cette fonction, le superviseur doit sélectionner un agent dans la liste puis cliquer sur le bouton ‘’ pour l'activer. Il doit à nouveau cliquer dessus pour désactiver la fonction.

Enregistrement

La fonction d'enregistrement permet au superviseur d'enregistrer la conversation d'un agent.

Pour utiliser cette fonction, le superviseur doit d'abord sélectionner un agent dans la liste, puis cliquer sur le bouton ‘’. Lorsque l'enregistrement est actif le bouton présente cette icône ‘’. Pour interrompre l'enregistrement, le superviseur doit cliquer à nouveau sur ce bouton.

Aperçu

Colonnes

Aperçu

Chaque ligne possède un ensemble de colonnes qui contiennent les informations relatives à la rubrique. L'objet de ce chapitre est de décrire les types de colonnes existants et les genres de paramètres qu'elles contiennent afin de permettre une meilleure compréhension de la signification des informations dans ces colonnes.

Le lecteur trouvera la description des colonnes individuelles de chaque vue au chapitre 'Description des colonnes' à la page 26.

Paramètres d'une colonne

Les principaux paramètres d'une colonne sont :

1.    Nom

2.    Description

3.    Type.

4.    Style.

5.    Visible

6.    Visible dans panneau d'information (voir section 'Zone panneau d'information')

7.    Type panneau d'information (voir section 'Zone panneau d'information'.

Le 'nom' de la colonne est utilisé dans l'en-tête pour décrire brièvement la signification de la colonne. A côté du nom, nous avons la 'description' de la colonne, qui fournit une description plus détaillée de la colonne et peut être visualisée dans le panneau d'information.

Chaque colonne possède un 'type' de paramètre. Ce paramètre indique les différentes 'valeurs' possibles dans une colonne (valeur graphique, liste d'ID, …). En plus de ces types, nous pouvons utiliser un formatage défini dans le paramètre 'style').

Le premier type est le 'type valeur'. Il s'agit d'une valeur fournie par la Nixxis Contact Suite, par exemple le 'nombre d'appels en attente'. Cette valeur ne peut pas être modifiée par le superviseur, mais elle peut être utilisée par le superviseur dans les 'colonnes définies par l'utilisateur'. Le second type est le "type graphique'. Il ne contient pas de données mais utilise des colonnes du premier type. Une colonne de ce type ressemble à ‘’. Chaque élément dans la barre graphique est représenté dans une couleur différente. Pour savoir ce que signifie la couleur, vous pouvez cliquer sur la colonne. Elle va alors apparaître dans la 'zone panneau d'information'. Le type suivant est le "type liste'. Ce type est une liste d'éléments représentés par leur identifiant. Par exemple, un agent peut être membre de plusieurs équipes. ‘Agent.Parameters.TeamId’ représente donc les identifiants de ces équipes. Lorsque le superviseur clique sur une colonne de ce type, plus d'informations apparaissent dans la 'zone panneau d'information' avec quelques informations supplémentaires concernant les identifiants.. Lorsque cette colonne s'affiche dans la vue principale, le superviseur ne voit pas la liste d'ID, mais c'est le nombre d'éléments dans la liste qui est affiché. Le dernier type est ‘type calcul'. Ce type est utilisé pour les 'colonnes définies par l'utilisateur'. Ce type ne contient donc aucune donnée réelle mais calcule sa valeur à partir d'une formule prédéfinie.

Les colonnes du type 'valeur' et 'calcul' possèdent un paramètre supplémentaire appelé ‘style’. Il existe quatre styles :

-      Texte : l'alignement se fait à gauche, pas d'autre formatage spécial.

-      Numérique : l'alignement se fait à droite, pas d'autre formatage spécial. Si la valeur n'est pas numérique ou un zéro, elle est réglée sur 0.

-      Durée : l'alignement se fait à droite. La valeur est formatée en ‘H:MM:SS’. La valeur saisie est une valeur numérique qui exprime la durée en millisecondes. Si cette valeur n'est pas numérique, elle est réglée sur 0. Par exemple : ‘3665000’ deviendra ’1:01:05’.

-      Pourcentage : l'alignement se fait à droite. La valeur est formatée en ‘0.00%’. Si la valeur saisie n'est pas numérique, elle sera réglée sur 0.

Le paramètre ‘visible’ indique si la colonne sera visible dans la vue principale par défaut ou non. Le paramètre ‘visible dans panneau d'information’ indique si la colonne doit être visible dans le panneau d'information ou dans le panneau détaillé du type (plus d'informations à la page 23).

Le dernier paramètre est 'type de panneau d'information'. Il indique le type d'informations qui doivent être affichées dans le panneau d'information. Voir chapitre suivant pour plus d'informations.

Zone panneau d'information

Aperçu

Le panneau d'information est utilisé pour afficher des informations supplémentaires concernant la colonne sélectionnée. L'affichage des informations sera spécifique en fonction de la configuration de la colonne (voir section ‘Colonnes’ à la page 21).

La zone panneau d'information se divise en trois sections. Dans le haut, on trouve le nom de la dernière colonne sélectionnée. En dessous, on trouve la description de cette colonne. La troisième et dernière partie affiche la zone d'informations avec le panneau. Voici les quatre types de panneaux :

  1. Panneau détails : montre une vue verticale des colonnes. Est également le panneau par défaut.
  2. Panneau barre graphique  : montre des informations concernant un type de graphique sélectionné.
  3. Panneau historique colonne : montre les pics historiques pour une colonne sélectionnée.
  4. Information sur les types de listes de colonnes : montre des informations concernant les ID dans une colonne du type de liste.

Les prochains chapitre expliquent ces différents panneaux en détail.

Panneau détails

Le 'panneau détails' montre une vue verticale de toutes les colonnes de la ligne sélectionnée en incluant celles qui ont été paramétrées sur invisibles avec l'option 'sélection de colonne’.

Ce panneau se compose de deux colonnes. La première contient les descriptions de la colonne, la seconde affiche la valeur courante de la colonne.

Panneau barre graphique

Le 'panneau barre graphique' fournit au superviseur des informations concernant la colonne du type graphique. Vous trouverez dans un tableau toutes les couleurs et la valeur qu'elles représentent.

La première colonne montre la couleur. La colonne à côté montre la valeur courante. La dernière colonne fournit le nom de cette colonne.

Panneau historique colonne

Le panneau suivant est le 'panneau historique colonne'. Ce panneau fournit au superviseur un bref historique des pics de valeur sur un intervalle de temps pour la colonne sélectionnée. Le pic est la plus grande valeur atteinte pendant un intervalle de temps. Ce graphique est une représentation graphique des pics par rapport à la liste complète. Cela signifie que la valeur la plus élevée représente 100% de la longueur totale du graphique et qu'elle apparaît en rouge. Les autres valeurs sont en rapport avec elle.

Description des colonnes

Aperçu

Dans ce chapitre, nous allons aborder la description et les réglages par défauts des colonnes (voir page 21 pour plus d'informations à propos des paramètres des colonnes).

Catégories

Le Superviseur Nixxis répartit les colonnes en différentes catégories. Le Superviseur Nixxis connaît les catégories suivantes :

-      Paramètres : contient essentiellement des colonnes ayant trait à la configuration ou à la description d'une rubrique. Par exemple : nom, description, mode de commutation…

-      Temps réel : nos colonnes concernant le statut actuel de la rubrique.

-      Historique : fournit un aperçu de ce qui s'est passé pendant une période de temps.

-      Colonnes systèmes : autres colonnes utilisées par le Superviseur Nixxis.

-      Colonnes personnalisées : ou ‘colonnes définies par l'utilisateur'. Colonnes créées par le superviseur. Voir chapitre 'Colonnes définies par l'utilisateur' à la page 16 pour plus d'informations.

Annexe

Modes de composition sortante

Il existe sept modes de composition différents pouvant être définies pour les activités sortantes :

-      CallbacksOnly : aucun nouvel enregistrement n'est composé; seuls les enregistrements précédemment composés et fermés en indiquant qu'ils devaient être rappelés à un moment donné sont composés.

-      Preview : les agents sont d'abord invités à confirmer l'enregistrement qui va être composé.

-      Progressive : une fois un appel terminé, l'appel suivant pour cet agent est composé. Les détails de l'enregistrement sont soumis à l'agent lorsque la communication a été établie.

-      Power : le système appelle plus d'une ligne par agent devenu disponible. Cela permet de limiter le temps entre deux appels.

-      Restricted Power : comme la composition "power", mais moins agressive.

-      Predictive : le système "prévoit" quand il doit commencer à composer et combien de lignes il a besoin afin de limiter le plus possible le temps entre deux conversations.

-      Unattended : le système appelle tous les numéros et joue l'IVR défini dans le champ pré-processeur à la section générale. En fonction de cet IVR, un appel peut être dirigé vers un agent ou non. Cette méthode est typiquement utilisée pour les activités de notification et confirmation.

Statuts des agents et actions des agent

Remarques introductives

Dans cette partie de la documentation, le concept de 'statut d'agent' sera clarifié à travers différents exemples et illustrations, afin de faciliter la compréhension du lexique utilisé dans cette documentation.

L'organigramme qui précède décrit le cycle de travail quotidien d'un agent de centre d'appels utilisant une application de centre d'appels. Pour être bref, voici les actions réalisées par un agent de centre d'appels :

-      L'agent se connecte à l'application du centre d'appels en saisissant son nom d'utilisateur et/ou son mot de passe.

-      Une fois connecté, il se trouve automatiquement en pause.

-      L'agent clique sur "prêt" et attend la réception d'un appel.

-      L'agent est connecté à un client et lui parle.

-      Lorsque la conversation est terminée et le client déconnecté, l'agent prend un moment pour effectuer le travail "d'après-appel" nécessaire. Cela signifie que l'agent complète l'historique du client et met à jour le dossier du client.

-      Une fois le script terminé, l'agent a trois options :

o    Il peut prendre un autre appel en cliquant à nouveau sur "prêt"

o    Il peut passer en pause

o    Il peut arrêter de travailler en se déconnectant.

Statut d'agent:

Le statut de l'agent utilisé dans le logiciel Nixxis est basé sur l'exemple ci-dessus. La définition d'un statut d'agent est "la situation globale d'un agent à un moment donné”.

Les statuts d'agent utilisés dans le logiciel sont :

-      En attente : l'agent attend d'être connecté à un contact.

-      En ligne : l'agent est en ligne avec un ou plusieurs contacts.

-      Conclusion : l'agent n'est pas connecté à un contact et effectue des actions d' "après-appel" (mise à jour du dossier et de l'historique du client)

-      Pause : l'agent est connecté au système mais ne travaille sur aucun contact et n'attend pas de recevoir d'appels du système.

La connexion et la déconnexion du système ne sont pas considérés comme des statuts d'agent, mais comme des événements. C'est la raison pour laquelle ils n'ont pas de durée.

Pour avoir une meilleure idée des statuts de l'agent, regardez les illustrations ci-dessous. Elles montrent une partie du cycle de travail quotidien d'un agent de centre d'appels.

1.    L'agent se connecte.

2.    Le statut de l'agent se trouve automatiquement sur "Pause".

3.    L'agent clique sur "Prêt".

4.    Le statut de l'agent passe sur "En attente". L'agent attend à présent un appel du système.

5.    Le statut de l'agent passe sur "En ligne" dès que ce dernier est connecté avec un client.

6.    Le client se déconnecte de l'agent.

7.    Le statut de l'agent passe alors sur "Conclusion" une fois le client parti.

8.    Une fois la conclusion effectuée, l'agent peut attendre un autre appel, passer en pause ou se déconnecter.

Actions de l'agent

Comme mentionné précédemment, les statuts d'agent représentent "la situation globale" d'un agent. Le statut de l'agent est dérivé du ou des contacts individuels de ce dernier.

Dans le système, chaque contact est divisé en ce que l'on appelle des "actions d'agent". Ces actions spécifient une certaine condition du contact pendant toute la procédure du contact en ligne.

Les actions d'agent dans le logiciel sont :

-      Composition en cours : cette action indique que le système essaie de connecter un contact avec l'agent.

-      En ligne : cette action indique que l'agent est en ligne avec un contact.

-      En attente : cette action indique que le contact a été mis en attente. Pendant ce temps, le contact n'est plus connecté à l'agent.

-      Conclusion : cette action indique que l'agent n'est plus connecté avec le contact et qu'il effectue les opérations d'après-appel de ce contact.

Le logiciel Nixxis permet à l'agent de traiter simultanément des contacts multiples.

Pour faciliter la compréhension, les exemples suivants vont illustrer différentes situations avec un et plusieurs contacts, avec chaque fois le lien entre les statuts de l'agent et les actions de l'agent.

Règles générales pour le traitement simultané de contacts multiples

La façon dont les statuts généraux de l'agent découlent des actions individuelles de l'agent avec ses contacts est basée sur ces règles :

-      - Si au moins un des contacts actifs est en ligne, le statut général de l'agent sera également en ligne.

-      - Si aucun contact n'est en ligne et au moins un contact est en conclusion, le statut général sera en conclusion.

-      - Si aucun contact n'est en ligne ou en conclusion et si l'agent n'est pas en pause, le statut général sera en attente.

Conclusion

Les concepts suivants doivent être parfaitement compris :

-      Un agent peut avoir des contacts (en ligne) multiples simultanément.

-      Le statut d'un agent indique "la situation globale d'un agent à un moment donné”.

-      Les statuts de l'agent découlent des (différentes) actions de l'agent.

-      Les actions de l'agent spécifient une certaine condition du contact pendant toute la procédure du contact en ligne.

-      Si un agent possède plus d'un contact actif simultanément, la manière dont le statut de l'agent est déterminé est basé sur des règles hiérarchiques entre les actions.
